Overview

My portfolio website was created to showcase my UX projects for potential employers and to highlight my UX design skills as well frontend skills.

This website was built using HTML, CSS and Javascript languages that resulted from the knowledge that I gained throughout the Frontend Development for Designer Course at Career Foundry. As a result, I develop important skills that not only made me confident as a designer, but as well in the contact with web development teams.

The final result is a responsive website that meets accessibility standards and sustainable hosted within a period of two months (March - May 2019)

You can access the GitHub repository for the website - Here

Context and Challenge

In addition to my UX design skills I was encourage to learn the basics of HTML, CSS and JavaScript languages. Therefore, I would enhance my skills and learn how to work better with other designers and/or web developers within the team.

As a prospect UX designer, showcase a strong portfolio is core for landing in the first UX job. Therefore, as I only had a pdf resume and a Behance page, building a portfolio site was crucial to show my work and development skills.

Overall, I aimed with this project to learn the basic of frontend development and have a responsive and accessible website that portrait me as UX designer.

Process and Insight

The idea of creating a website from scratch using frontend languages sounded terrifying. I guess this is the feeling when one is trying something new. However this time I could count with a lot of support from my mentor.

The course project brief suggested a home, about, projects and contact pages and provided low-fidelity wireframes of what the pages would look like and what content would be included.

I then wrote all the content of my portfolio and made font and color palette choices. As I wanted the website to portrait who I am as a person and designer, colours related with nature and clear and simple layout design were vital. Hence, reflecting minimalism, simplicity and quietness. I wanted the design to reflect professionalism and sophistication through the use of black in a clean white background. The use of green is to evoke sustainability and compassionate design, as well to emphasize balance and stability. With this combination of contrasting colors, I intend to create a welcoming, and enjoyable experience for the user throughout the portfolio site. I have chosen to use the typography of Roboto as it is portfolio friendly, clean and it allows users to read comfortably. I have also looked through several professional UX designers’ websites for ideas and style inspiration.

Following this, I started translating the initial wireframes layout through HTML, identifying the head, body, header, hero and footer of the pages, as applicable. By placing "div" container throughout these sections, I was able to customize the structure of my website. I added the content to each section, from text, pdf’s to optimized images and made the respective connections between the pages. Then I started using CSS to style the pages, being able to customize how visually I wanted the website to look like. I also used media queries that enable me to create break points on different screen sizes and style my pages using the mobile-first approach. Additionally, I was able to animate an image using keyframes in CSS animation.

The learning process of these two languages was challenging, as it required a lot of focus, organization, systematic and logical thinking. The positive aspect was the fact that I could see straightaway the result of my inputs and change it as required. Using HTML and CSS allowed me to understand better how flexible the layout can be and how to translate wireframes through code. Consequently, this completely changed my way of wireframing and prototyping as a designer. However, JavaScript was harder to comprehend and I required more support from my mentor and research. Alongside, I also gained knowledge how to use GitHub, create repositories and define commits.

As a result, I finished with a fully functioning responsive website that was ready for user testing. The website prototype was tested on 5 participants with background that range from software developers to UX designers. Overall, all the participants gave a positive feedback to the website, specially the simple, clear and straightforward layout and navigation.

Solutions

With the feedback of the user testing, iterations were made:

  • Reduce of the amount of text on the hero section and description of the projects;
  • More images were added in the description of the project;
  • “Go to top” removed from home and about pages;
  • Fixed hover and active color for hamburger menu icon;
  • Fixed the speed of the scroll down;
  • Changed profile picture
  • Adjustments of padding and margins throughout the all website;
  • Rewording of the “Contact” section

Besides to all these adjustments, the website was tested for accessibility guidelines, run through code linters and cross-browser testing to ensure code quality. There were some minor code errors that were easily fixed; as well the background color in the header section was replaced by #308373 to meet the AA accessibility guidelines.

Results

This website has now been published and is working well across multiple platforms and screen sizes.
